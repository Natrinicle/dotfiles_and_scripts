# Communication Pattern Analysis

Single writer for `{contact}-communication-pattern-analysis.md` under project or
global memory. Other skills **invoke** this skill; they must not freestyle those files.

## When

- `contact-syncer` / inbox / adversarial flows pass evidence of recurring friction  
- User asks to document how a relationship tends to go wrong  

## Inputs from caller

Contact name, source (chat/mail/ticket/meeting), timestamped quotes from both
sides, pattern labels, optional severity.

## Patterns (intent)

- Compliance stated but not registered, then re-asked  
- Silence on direct questions beyond a reasonable window  
- Goalpost movement after delivery  
- Loaded or status-laden language  
- Channel thrash (promise in chat, no follow-through in PR/ticket)  

Stick to observables and quotes. No amateur diagnosis.

## File habits

- Append dated episodes; keep a short “standing patterns” summary at top  
- Link cross-channel evidence when a redirect was claimed  
- When patterns clear, note resolution rather than deleting history
