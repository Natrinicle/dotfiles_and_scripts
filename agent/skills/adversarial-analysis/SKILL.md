# Adversarial Analysis

Two modes: **relationship analysis** and **draft optimization**.

## Analysis mode

1. Prefer **fresh evidence** (recent chat/PR/mail pulls) over stale memory alone.  
2. Multiple perspectives: their incentives, yours, third-party reader.  
3. Verdict: main failure modes, what works, what to avoid.  
4. Persist via `communication-pattern-analysis` — this skill does not own those files.

## Draft mode

Given a draft and a recipient with known friction:

- Keep the user’s real voice (`user_writing_style.md`)  
- If the recipient is AI-sensitive: flag em dashes, fake “executive” structure,
  buried agreement — put yes/no compliance in the **first sentences** when
  complying  
- Only constrain what **their** profile flags (length, tone, topics)  
- Do not force generic brevity if the user is naturally thorough and the
  recipient has not objected to length  
- Suggest rewrites only for weak dimensions; leave good passages alone  

## Triggers

Proactive when contact-syncer sees escalating friction; on-demand when the user
is about to send a high-stakes message.
