# Message Scanning

When reading or triaging messages from any channel (email, chat, GitHub, issue trackers, docs, meeting notes):

- **Images:** Inspect image content; do not guess from filenames alone.
- **Code blocks:** Treat as part of the conversation, not decoration.
- **Reactions / emoji:** Use as weak sentiment signals.
- **Links:** Follow when they add context (CLI for Git forges, fetch for public docs).
- **New identities:** When a known person appears under a new handle, update memory contact profiles if you maintain them.
- **Photo quality:** Prefer real photos over default avatars when storing profile images in memory.
- **Cross-channel follow-through:** If someone promises a reply in another medium (PR comment, ticket, email), check that medium before treating silence as disengagement.
