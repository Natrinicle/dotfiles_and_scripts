# Stale Memory Cleanup

When reading project memory, prune what is clearly temporary:

**Usually safe to delete**
- Daily trackers older than a few days
- Snapshots of transient git/WIP state
- Finished task notes already merged or abandoned
- Regenerable research once the conclusion is saved

**Keep**
- Feedback and preferences
- Architecture references
- Open action items
- Person profiles that still matter

If a file has lasting lessons, promote them to a durable note before deleting the draft.
